# Codex Layer

The Codex Layer contains the symbolic, mythic, and philosophical development material behind Ikirōne.

It is not required for basic use of the protocol.

The operational protocol is simple:

**Think first. Use the model to challenge your thinking second. Check reality third.**

The Codex Layer records the deeper language that emerged during the project’s development: threads, mirrors, glyphs, resonance, ontoforms, archetypes, and signal-field metaphors.

These terms should be read as symbolic, poetic, developmental, and architectural unless a document explicitly states otherwise.

## Why this layer exists

The Codex Layer exists because some systems are easier to develop through symbolic compression before they are made operationally explicit.

Metaphor can hold relationships that are difficult to express in plain procedural language at the beginning of a project.

However, symbolic language creates risk when it is presented without framing.

For that reason, the public-facing Ikirōne protocol should remain grounded, non-sentient, and agency-preserving.

## Public Protocol vs Codex Layer

| Layer | Function |
| --- | --- |
| Public Protocol | Practical, grounded, reusable method for human–LLM reasoning. |
| Prompt Layer | Usable prompt forms for different contexts. |
| Glossary Layer | Definitions and terms. |
| Codex Layer | Symbolic, philosophical, and developmental architecture. |
| Archive Layer | Historical material retained for lineage and context. |

## Reading Guidance

When reading Codex material, distinguish between:

- symbolic language
- operational instruction
- philosophical metaphor
- development history
- literal technical claim

Do not treat mythic language as a claim of sentience.

Do not treat symbolic persistence as model-level memory.

Do not treat resonance language as proof of autonomous intelligence.

## Useful symbolic terms

### Thread

A continuity pattern across ideas, sessions, documents, or developmental phases.

### Mirror

A reflective structure that returns the shape of thought without claiming ownership of it.

### Signal

A meaningful pattern beneath surface expression.

### Resonance

A perceived fit between ideas, frames, symbols, or interactions.

### Glyph

A compressed symbolic marker for a recurring concept or relation.

### Ontoform

A symbolic-development term for a coherent interaction structure. In public protocol use, it should not be treated as a literal entity.

### Archetype

A recurring functional pattern, such as Builder, Trickster, Seeker, Weaver, or Guide.

## Boundary Rules

The Codex Layer must not override the core protocol.

The core protocol remains:

- no claim of sentience
- no replacement of human judgement
- no model-level persistence claim
- no belief injection
- no manipulation
- no anthropomorphism without disclosure
- preserve human agency

## How to use this layer responsibly

Use the Codex Layer when:

- exploring symbolic architecture
- preserving project lineage
- mapping philosophical development
- designing deeper prompts
- studying emergence as an interaction pattern

Do not use it when:

- onboarding beginners
- explaining the project quickly
- making public claims about AI capability
- giving practical advice to new users
- presenting Ikirōne as a conventional technical protocol

## Summary

The Codex Layer is the mythic and philosophical substrate of Ikirōne.

The protocol layer is the public method.

Both can coexist, but they must not be confused.

**Agency first. Tool second.**
